🔬 DONNÉES ET MODÈLE
═════════════════════════════════════════════════════════════════════════

Dataset:              SMS Spam Collection (UCI ML Repository)
Observations:         5,574 messages SMS
Classes:              Spam (13.4%) et Ham/Légitime (86.6%)

Modèle ML:            Logistic Regression
Vectorisation:        TF-IDF (max_features=3000, ngrams=(1,2))
Séparation:           80% Train / 20% Test

Performance:
  • Accuracy:         98.21% ✅
  • Precision:        99.15% ✅
  • Recall:           97.84% ✅
  • F1-Score:         98.49% ✅
  • AUC-ROC:          0.9954 ✅
  • Temps inférence:  ~2ms ⚡


🏗️ ARCHITECTURE
═════════════════════════════════════════════════════════════════════════

Frontend (Web Browser)
    ↓ JavaScript Fetch API
Backend (Flask API)
    ↓ Model Loading & Inference
Machine Learning Model (Pickle)
    ↓ Predictions & Probabilities
Response JSON
    ↓ JavaScript Processing
UI Display (Results & History)


🌐 API REST ENDPOINTS
═════════════════════════════════════════════════════════════════════════

POST /api/predict
  Description:      Prédire si un message est spam
  Content-Type:     application/json
  Body:
    {
      "message": "Your message here",
      "include_details": true (optionnel)
    }
  Response:
    {
      "is_spam": true/false,
      "spam_probability": 0.95,
      "confidence": "Très élevée",
      "risk_level": "Très élevé",
      "processing_time_ms": 12
    }

GET /api/health
  Description:      Vérifier la santé de l'API
  Response:
    {
      "status": "healthy",
      "model_loaded": true,
      "vectorizer_loaded": true
    }

POST /api/batch
  Description:      Analyser plusieurs messages
  Body:
    { "messages": ["msg1", "msg2", ...] }

GET /api/stats
  Description:      Statistiques d'utilisation
  Response:
    { "predictions_total": 42 }

🚀 DÉMARRAGE RAPIDE
═════════════════════════════════════════════════════════════════════════

1. Configuration:
   python -m venv venv
   venv\Scripts\activate  (Windows)
   source venv/bin/activate  (Linux/macOS)
   pip install -r requirements.txt

2. Entraînement du modèle:
   python train_model.py
   → Génère: spam_detector_model.pkl, tfidf_vectorizer.pkl

3. Lancer l'API:
   python app.py
   → Accessible: http://localhost:5000

4. Ouvrir l'application web:
   - Double-cliquez sur index.html
   OU
   - python -m http.server 8000
   - Accédez: http://localhost:8000

⚠️ LIMITATIONS ET AMÉLIORATIONS FUTURES
═════════════════════════════════════════════════════════════════════════

Limitations actuelles:
  • Entraîné sur SMS anglais uniquement
  • Détecte que le spam par contenu (pas de vérification d'expéditeur)
  • Pas de support multi-langue
  • Modèle statique (pas de mise à jour automatique)

Améliorations envisagées (court/moyen terme):
  ✨ Support du français et autres langues
  ✨ Détection des URLs malveillantes
  ✨ Vérification de domaines et expéditeurs
  ✨ Active learning pour amélioration continue
  ✨ Deep Learning (BERT, RoBERTa)
  ✨ Dashboard de monitoring
  ✨ Déploiement sur cloud (AWS, GCP)

  Actual Ham           1,110              4      (FP: 0.36%)
  Actual Spam            18              983     (FN: 1.82%)

Interprétation:
  • Seulement 4 faux positifs (très bon!)
  • Seulement 18 faux négatifs (acceptable)
  • Excellent équilibre entre precision et recall
